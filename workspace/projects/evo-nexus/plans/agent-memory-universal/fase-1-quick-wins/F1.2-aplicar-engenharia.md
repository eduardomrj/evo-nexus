# F1.2 — Aplicar o protocolo aos agentes de engenharia

**Tipo:** [ATIVAR] (texto em prompts existentes)
**Agentes envolvidos:** @bolt-executor (alvo: prompts de Bolt, Apex, Hawk, Lens, Grid, Flow, Oath)
**Complexidade:** LOW

## Descrição

Os agentes de engenharia hoje só têm um stub de memória ("leia sua pasta primeiro" + "read access a `memory/`"). Adicionar o bloco canônico de F1.1 às system prompts deles, transformando memória de "só quando o usuário pede" em "salvar proativamente durante a sessão". Esta é a Camada A — e resolve a maior parte do problema sozinha.

## Escopo exato

Aplicar a: **Bolt, Apex, Hawk, Lens, Grid, Flow, Oath** (7 agentes).
Compass já tem o protocolo — fica de fora, serve como referência viva.

## Passos concretos

1. Para cada um dos 7 agentes em `.claude/agents/{slug}.md`:
   - Localizar a seção de memória existente (ex: Bolt linha 25 "read access" + linha 92 "read your memory folder first").
   - Promover de "read access" para "**read and write access**" a `memory/` quando aplicável ao papel (engenharia escreve em `memory/projects/` ao tocar área nova).
   - Inserir o bloco completo de F1.1 com `{slug}` substituído pelo slug real.
2. Ajustar o gatilho de salvamento ao papel de cada agente:
   - **Bolt/Grid/Flow:** salvar `feedback` (correções de approach, convenções confirmadas) e `project` (gotchas de build/test).
   - **Apex/Lens/Oath:** salvar `project` (decisões de arquitetura, critérios de review/verificação validados) e `feedback`.
   - **Hawk:** salvar `feedback` (padrões de debugging que funcionaram) e `project` (causas-raiz recorrentes) — sem virar "fix recipe", que é proibido.
3. Garantir que cada bloco aponta para `/home/evonexus/evo-nexus/.claude/agent-memory/{slug}/`.

## Acceptance criteria

- `grep -c "Persistent Agent Memory" .claude/agents/{bolt,apex,hawk,lens,grid,flow,oath}*.md` retorna ≥1 para cada.
- Nenhum bloco aponta para `/Users/etus_0104/...`.
- O protocolo respeita a exclusão "não salvar código/git history/fix recipes" do CLAUDE.md.
