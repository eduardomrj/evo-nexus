**Tipo:** [CONSTRUIR NOVO] (artefato de referência reutilizável)
**Agentes envolvidos:** @bolt-executor
**Complexidade:** LOW

## Descrição

Existe hoje um bloco de protocolo de memória idêntico em Oracle e em todos os agentes de negócio (e em Compass). Antes de propagá-lo para os agentes de engenharia, extrair uma **versão canônica única** desse bloco como template, parametrizada por `{slug}`, para garantir consistência e facilitar manutenção futura.

## Passos concretos

1. Ler o bloco `# Persistent Agent Memory` de um agente de negócio (ex: `clawdia-assistant.md`, linhas ~157–294) — é a versão completa de referência.
2. Comparar com o bloco já presente no Compass (`compass-planner.md`) e no Oracle para confirmar que são equivalentes.
3. Criar template canônico em `.claude/templates/agent-memory-protocol.md` com:
   - Tipos de memória (`user` / `feedback` / `project` / `reference`) e quando salvar cada um.
   - O que NÃO salvar (código, git history, fix recipes, conteúdo de CLAUDE.md, estado efêmero).
   - Estrutura de arquivo com frontmatter (`name`, `description`, `metadata.type`).
   - Disciplina de `MEMORY.md` (índice de uma linha, nunca conteúdo).
   - Regras de não duplicar e verificar antes de recomendar.
   - Placeholder `{AGENT_MEMORY_PATH}` = `/home/evonexus/evo-nexus/.claude/agent-memory/{slug}/`.
4. Adaptar o tom para engenharia: ênfase em `feedback` (correções/confirmações de approach) e `project` (decisões de arquitetura, gotchas), menos em `user`.

## Acceptance criteria

- `.claude/templates/agent-memory-protocol.md` existe e usa o caminho deste ambiente (não `/Users/etus_0104/...`).
- O template é textualmente equivalente ao protocolo do Oracle/Clawdia, só parametrizado por `{slug}`.
