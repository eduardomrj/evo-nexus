# F3.2 — Extração de recuperação para quem não salvou

**Tipo:** [EVOLUIR]
**Agentes envolvidos:** @bolt-executor, @clawdia-assistant (EOD)
**Complexidade:** MEDIUM

## Descrição

Fechar o ciclo: para cada agente que esteve ativo no dia (mapa de F3.1) mas **não** tem memória nova daquela data — nem salvou proativamente (Camada A) nem foi capturado pela rede de segurança (Camada B) — rodar a extração de recuperação reaproveitando o extrator de F2.2. É a terceira e última rede.

## Passos concretos

1. Para cada agente no mapa de F3.1:
   - Verificar se há arquivo novo / `MEMORY.md` atualizado em `.claude/agent-memory/{slug}/` com data de hoje.
   - Se SIM (A ou B cobriram) → pular.
   - Se NÃO → rodar `extract(slug, transcript)` de F2.2 sobre os JSONLs do dia daquele agente.
2. Reusar integralmente o gate de privacidade e a dedup de F2.2 — não reimplementar.
3. Rodar dentro do fluxo do EOD (21:00), depois da coleta e antes de gerar o daily log, para que o log já reflita a memória recuperada.
4. Respeitar limite de custo: extração em lote no EOD pode tocar vários agentes; logar custo total.

## Acceptance criteria

- Agente ativo no dia sem memória nova → ganha entradas via extração de recuperação.
- Agente que já salvou (A ou B) → não é reprocessado (sem duplicação).
- Sessões sensíveis puladas pelo mesmo gate de F2.2.

## Dependências

F3.1 (mapa de agentes) + F2.2 (extrator, reutilizado).

## Riscos

- MÉDIO: custo de processar muitas sessões no EOD. Mitigar pulando quem já salvou e limitando ao escopo de agentes.
- BAIXO: re-extrair conteúdo já capturado pela Camada B no mesmo dia. A dedup de F2.2 cobre isso.
