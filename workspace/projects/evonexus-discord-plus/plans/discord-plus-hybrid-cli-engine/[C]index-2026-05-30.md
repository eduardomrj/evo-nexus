# Work Plan — Discord Plus modo híbrido C: CLI OpenClaude por thread/canal

## Context

O Discord Plus já está operacional com gateway próprio, policy, threads e replies seguros, mas o SDK v2 não entrega agente principal real; prompt Oracle não basta. A decisão de produto é implementar um modo híbrido C: manter o Plus como gateway/policy/reply e executar sessões por canal/thread via CLI pública `openclaude --agent oracle` ou agente configurável, sem tocar no core OpenClaude.

## Objectives

- Criar um engine CLI configurável para sessões do Discord Plus, preservando o engine SDK atual como rollback.
- Executar agente real via CLI pública do OpenClaude, inicialmente Oracle, com agente configurável por env.
- Manter isolamento por `sessionKey` de canal/thread/DM e continuidade quando a CLI expuser mecanismo público.
- Garantir que replies continuem saindo pelo gateway seguro do Plus, sem plugin Discord oficial ativo duplicado.

## Guardrails

### Must Have

- Configuração explícita de engine, ex. `EVONEXUS_DISCORD_PLUS_SESSION_ENGINE=cli`.
- Configuração explícita de agente padrão, ex. `EVONEXUS_DISCORD_PLUS_DEFAULT_AGENT=oracle`.
- `SessionSupervisor` deve continuar controlando `sessionKey`, limites, TTL e restart por sessão.
- O runner CLI deve usar spawn/exec com array de argumentos, sem interpolação shell.
- Policy deve ser aplicada antes de chamar a CLI e revalidada antes de side effects reais.
- Testes devem cobrir engine selection, montagem de comando, session key, fallback/rollback e bloqueio de gateway duplicado.

### Must NOT Have

- Não alterar `/home/evonexus/evo-nexus` core OpenClaude nem dependências internas do core para viabilizar o recurso.
- Não usar `--channels plugin:discord@...` na CLI ou iniciar segundo gateway Discord.
- Não ampliar permissões Discord, intents, allowlists ou policy.
- Não substituir o mecanismo seguro de reply do Plus por side effect direto da CLI/plugin oficial.
- Não inventar flags da CLI: flags de continuidade e modo não interativo entram como investigação antes de codificar contrato final.

## Success Criteria

- [ ] `EVONEXUS_DISCORD_PLUS_SESSION_ENGINE=cli` usa CLI pública do OpenClaude sem alterar core.
- [ ] `EVONEXUS_DISCORD_PLUS_DEFAULT_AGENT=oracle` produz sessão com agente real Oracle ou registra falha factual se a CLI não suportar a flag esperada.
- [ ] Gateway Discord permanece único; nenhum `--channels plugin:discord@...` é usado pelo runner CLI.
- [ ] Cada canal/thread/DM mantém `sessionKey` isolada.
- [ ] Continuidade usa mecanismo público confirmado ou fallback documentado, sem flags inventadas.
- [ ] Replies passam pelo mecanismo seguro do Plus com policy revalidada.
- [ ] Rollback para SDK/legado é simples por env e sem migração destrutiva.
